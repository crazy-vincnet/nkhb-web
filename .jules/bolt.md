## 2024-05-24 - [React Component Re-render Optimization via React.memo and useCallback]
**Learning:** In this project's TypeScript environment, combining `React.memo` with the `React.FC` type annotation leads to compilation errors because `React.memo` returns a `React.NamedExoticComponent` instead of a standard function type. Also, modal triggers managed at the root `App.tsx` level (using `window.postMessage`) are passed deeply into child section components in `Home.tsx`. Since these handlers were originally inline arrow functions, they forced unnecessary re-renders of heavy UI components (like `Background`, `Composition`, and `Guide`) every time the `Home` component re-rendered (e.g. from route hash location changes).
**Action:** When memoizing static child components that receive callbacks, wrap the handler functions with `useCallback` in the parent, and use `React.memo` on the child while removing the `React.FC` annotation and applying inline prop typing.
