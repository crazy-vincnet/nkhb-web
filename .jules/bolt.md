## 2024-05-18 - [Root-level modal states triggering full-page re-renders]
**Learning:** In the React application (`src/public`), modal states (e.g., `isArticleModalOpen`, `isLetterModalOpen`, `isSampleModalOpen`) are managed at the root `App.tsx` level and can be triggered via `window.postMessage` events from deeply nested child components. This architectural pattern makes optimizing component re-renders critical, as every modal toggle otherwise causes a full re-render of the entire page hierarchy (including complex sections like `Background`, `Composition`, and `Guide`).
**Action:** When working on this specific codebase, always ensure that event handler props passed to section components are wrapped with `useCallback` and apply `React.memo` to static child section components to prevent these unnecessary and costly re-renders. Avoid `React.FC` when wrapping components with `React.memo` due to strict TS typing rules.
