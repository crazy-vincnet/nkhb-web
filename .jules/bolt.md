## 2024-05-24 - React Context/Modal Prop Drilling Performance
**Learning:** The application manages global modal state (`isArticleModalOpen`, etc.) at the very top level in `App.tsx` and triggers changes using global `window.postMessage` events instead of React Context. Because `App.tsx` re-renders when a modal opens, any deeply nested component that does not have `React.memo` and stable callbacks applied will unnecessarily re-render, creating a performance bottleneck for heavy layout components on pages like `Home`. The project architecture specifically requires `React.memo` to be used cautiously on large static sections to mitigate this App-level state management structure.
**Action:** When working on heavily nested page structures in this application, actively hunt for large static or semi-static sections and wrap them in `React.memo`, being careful to ensure any passed callback functions are stabilized with `useCallback` to prevent breaking the memoization during App-level state changes.
