## 2025-02-12 - Prevented full page re-renders on route navigation due to root-level modal states
**Learning:** In this application, modal states (like `isArticleModalOpen`) are managed at the top level in `App.tsx` and triggered via global `window.postMessage` events dispatched from nested components. If the `Home` page doesn't memoize these trigger functions (`onOpenArticle` etc.) using `useCallback`, every render of `Home` creates new function references. When combined with passing these props to unmemoized complex static child components (`Background`, `Composition`, `Guide`), this architecture inherently causes massive, unnecessary component tree re-renders.
**Action:** Always memoize event handler props (`useCallback`) and static child sections (`React.memo`) when root-level state triggers frequent re-renders down the tree. Also, remember not to use `React.FC` when defining components wrapped with `React.memo` to avoid TypeScript compilation errors.
