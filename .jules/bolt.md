## 2026-05-31 - [React Component Re-renders due to Global Modal State]
**Learning:** In this application, modal states (like `isArticleModalOpen`) are maintained at the root `App.tsx` level and triggered via `window.postMessage`. Because heavy section components in `Home.tsx` were passing inline arrow functions down as props, every modal open/close action caused a full cascade re-render of all static sections on the page. This pattern amplifies rendering cost significantly.
**Action:** When working on this specific architecture, ensure that global event triggers injected as props from layout/page components to large static sections are always wrapped in `useCallback` and the target functional components are wrapped with `React.memo` (without the incompatible `React.FC` typing).
