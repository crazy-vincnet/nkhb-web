## 2024-05-24 - [Avoid Inline Functions for Props Passed to Memoized Page Layout Components]
**Learning:** Passing inline arrow functions as props (e.g., `props.onOpenArticle = () => window.postMessage(...)`) within a page layout's render cycle dynamically creates new function references on every parent render. This defeats the purpose of wrapping child components (like `Background`, `Composition`, `Guide`) with `React.memo`, leading to unnecessary full-page re-renders. This is particularly problematic in applications like NKHB where state-driven modals and complex layouts co-exist.
**Action:** Always wrap event handlers in `useCallback` when passing them down to static or purely presentational section components within a dynamic page layout, and ensure those target child components are wrapped in `React.memo`.
