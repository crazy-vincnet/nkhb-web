
## 2024-05-18 - [Optimization of Top-Level React Handlers Impacting Static Sections]
**Learning:** In a single-page React application where modals are managed globally and triggered by deeply nested components using `window.postMessage`, avoiding unnecessary re-renders in heavy static sections is critical. I found that components receiving function props for event handlers will constantly re-render unless the functions are memoized with `useCallback` at their origin and the child components are wrapped in `React.memo`. Due to TypeScript compilation strictness, assigning a `React.memo` return value to a variable typed with `React.FC` causes errors; inline typing of props in the functional arguments must be used instead.
**Action:** When passing function handlers down to static or read-heavy child components, always wrap them in `useCallback` at the top level and ensure the children are wrapped in `React.memo` using inline typing to maximize rendering efficiency and satisfy TypeScript.
