## 2024-06-12 - Prevent root-level modal state from re-rendering all static section components
**Learning:** In the React application, modal states (e.g., `isArticleModalOpen`, `isSampleModalOpen`, `isLetterModalOpen`) are managed at the root `App.tsx` level and triggered via `window.postMessage` from deeply nested components. Every time a modal is opened or closed, it causes a full-page re-render. Static sections like `Background`, `Composition`, and `Guide` receive props containing inline arrow functions (e.g. `onOpenArticle = () => ...`). This defeats React's default bailout logic because the function references change on every render of `Home.tsx`.
**Action:** When components trigger global modal state managed by a root-level component via props, ensure the callback props are wrapped in `useCallback` in the parent and the static child section components are wrapped in `React.memo` to prevent cascading, unnecessary re-renders.
