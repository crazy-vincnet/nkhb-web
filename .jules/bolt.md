## 2024-05-24 - Modal State Architecture Performance Pattern
**Learning:** In this React app, modal visibility state is lifted to the root `App.tsx` and triggered via `window.postMessage`. Because of this architecture, opening any modal triggers a re-render of `App.tsx`, which cascades down through React Router to the page component (e.g., `Home.tsx`), and subsequently to all child section components. Passing anonymous functions directly in the `map` loop inside `Home.tsx` prevents React from bailing out of re-renders for memoized children.
**Action:** Always wrap event handlers that trigger `postMessage` with `useCallback` at the page level, and ensure static child sections (like `Background`, `Composition`, `Guide`) are wrapped with `React.memo` without explicit `React.FC` typing to satisfy Vite/TypeScript compilation.
