## 2023-10-27 - Root-level Modal State Causing Cascading Re-renders
**Learning:** In the `src/public` React application, modal states (`isArticleModalOpen`, etc.) are managed at the root `App.tsx` level and triggered via `window.postMessage`. Every time a modal opens/closes, `App.tsx` re-renders, causing `Home.tsx` and all its deeply nested child section components (like `Background.tsx`, `Composition.tsx`, `Guide.tsx`) to re-render. Passing inline functions as event handlers to these sections forces them to re-render even if they are wrapped in `React.memo()`. Furthermore, typing functional components as `React.FC` prevents successful wrapping with `React.memo()` due to TypeScript compilation issues in this specific environment.
**Action:** When managing root-level state that affects deeply nested layout components, always wrap event handler props in `useCallback` inside the parent (e.g., `Home.tsx`). In the child components, remove the `React.FC` type annotation, type the props inline, and wrap the export with `React.memo()` to prevent unnecessary DOM updates.
