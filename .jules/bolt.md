## 2024-05-30 - Top-level Modal State Management Causing Full-page Re-renders
**Learning:** In the React application architecture (`App.tsx`), modal states (e.g., `isArticleModalOpen`) are managed at the root component level and triggered via `window.postMessage`. This architecture causes the entire application tree, including static section components inside `Home.tsx`, to re-render whenever a modal is toggled. Passing inline arrow functions (e.g., `() => window.postMessage(...)`) to these child components exacerbates the issue by recreating function references on every render, preventing React's reconciliation algorithm from bailing out of updates for those children.
**Action:** When working with components that receive callback props triggered by global state changes, always wrap the callbacks with `useCallback` in the parent component and apply `React.memo` to the child functional components. Note: When typing memoized functional components in this environment, remove `React.FC` to prevent TS compilation errors with the `React.NamedExoticComponent` object returned by `React.memo`.
