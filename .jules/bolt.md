## 2024-05-28 - React.memo with TypeScript and App-Level Modal State Management
**Learning:** In a codebase where modal states are managed at the root `App.tsx` level and triggered by child components via `window.postMessage`, any state change in `App.tsx` will cause all child components mapped out in the Home page to re-render unnecessarily. When fixing this by wrapping the functional components with `React.memo` to skip re-renders, it's important not to use the `React.FC` type annotation on the variable assignment in TypeScript. `React.FC` types a function signature, whereas `React.memo` returns a `React.NamedExoticComponent` object, causing a TS compilation error. The correct approach is to type the props inline within the component arguments and allow TypeScript to infer the variable type.
**Action:** When wrapping components in `React.memo` for this codebase, remove the `React.FC` typing and define prop types inline. Additionally, ensure callback functions passed as props to these memoized components are wrapped in `useCallback` to maintain reference stability and prevent unwanted re-renders when parent state updates.
