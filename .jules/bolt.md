## 2023-11-09 - Memoizing global event handlers and static child components
**Learning:** In the React application (`src/public`), modal states (e.g., `isArticleModalOpen`, `isSampleModalOpen`, `isLetterModalOpen`) are managed at the root `App.tsx` level and can be triggered via `window.postMessage` events from deeply nested child components. To prevent unnecessary full-page re-renders of heavy, static section components caused by page-level state or routing changes (like URL hash changes for smooth scrolling), it is critical to wrap event handler props with `useCallback` at the parent level (`Home.tsx`) before passing them down, and then applying `React.memo` to the child components (e.g. `Background.tsx`, `Composition.tsx`, `Guide.tsx`). Otherwise, components recreate the function props on every render, defeating the purpose of `React.memo`.
**Action:** When working on parent components that pass down functions to static, heavy UI sections, proactively apply `useCallback` to the handlers and `React.memo` to the children to maintain optimal rendering performance in this centralized state architecture.
