## 2024-06-25 - Root-Level Modal State Causes Cascade Re-renders
**Learning:** In the current architecture of this React application, modal states (e.g., `isArticleModalOpen`) are managed at the root `App.tsx` level. These state changes trigger a re-render of `App.tsx`, which subsequently causes a re-render of nested page components like `Home.tsx` and all of their section child components (`Background.tsx`, `Composition.tsx`, `Guide.tsx`), even though these sections only present static content. The use of inline arrow functions for event handlers in `Home.tsx` exacerbated this by failing referential equality checks on every render.
**Action:** Always wrap event handler props (that are passed down to child components) with `useCallback` when they depend on stable references (like `window.postMessage`). Additionally, wrap purely presentational or static child components with `React.memo` so they only re-render if their props actually change. When applying `React.memo` in a TypeScript project, remove the `React.FC` type annotation from the component variable and use inline prop typing to prevent TS compilation errors.
