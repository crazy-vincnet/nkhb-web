## 2024-06-14 - React Modal Event Bus Optimization
**Learning:** In the React app, modaling states are managed globally in App.tsx via `window.postMessage` events triggered by child components deeply nested (e.g., in `Home.tsx` rendering `Background.tsx`). Passing inline arrow functions for these callbacks inside rendering loops of mapped components (e.g. `HOME_DEFAULT_LAYOUT` in `Home.tsx`) creates new function references on every render, and without `React.memo` on the child sections, it leads to full re-renders of the entire `Home` page hierarchy whenever the App state (modal visibility) changes.
**Action:** When working with page-level components that map over a layout array, wrap callback handlers passed down to them with `useCallback`, and ensure the child section components are wrapped in `React.memo` (without the `React.FC` type which conflicts in this TS setup). This specifically applies to the `onOpenArticle`, `onOpenSample`, and `onOpenLetter` props in the `Home` section mapped components.
