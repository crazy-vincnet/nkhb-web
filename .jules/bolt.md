## 2024-05-19 - [Prevent Unnecessary Re-renders from Root Modal State]
**Learning:** In this project, modal states (e.g., `isArticleModalOpen`, `isLetterModalOpen`, `isSampleModalOpen`) are managed at the root `App.tsx` level and triggered via `window.postMessage` events from deeply nested child components. This architectural pattern means that anytime a modal opens or closes, `App.tsx` re-renders, causing a cascading re-render of all static page sections (like `Home.tsx` and its children). Since `Home.tsx` dynamically passes down event handlers created inside the render loop, it broke child memoization.
**Action:** When working with page-level section components that dispatch root events in this codebase, ensure event handler props are wrapped in `React.useCallback` and the static child section components (`Background`, `Composition`, `Guide`) are wrapped with `React.memo` to optimize rendering performance.
