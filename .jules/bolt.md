## 2024-06-25 - Root-Level Modal Architecture Causing Cascading Re-renders
**Learning:** In this application, modal visibility states (e.g., `isArticleModalOpen`, `isSampleModalOpen`) are managed at the root `App.tsx` level and triggered via global `window.postMessage` events from deeply nested children. Previously, inline arrow functions were passed as props to layout components in `Home.tsx` to handle these messages. When `App.tsx` re-rendered (e.g. state change or context update), it caused `Home.tsx` to re-render, creating new arrow function references. Because `Background`, `Composition`, and `Guide` components were not memoized and were receiving these new function references, they re-rendered unnecessarily every time.
**Action:** When working with root-level state that cascades down via props or contexts, ensure handler functions are memoized with `useCallback` and wrap static child sections that receive these handlers with `React.memo()`. When using `React.memo()` in this TS environment, do not use `React.FC` on the variable assignment; type the props directly in the component's arguments.
