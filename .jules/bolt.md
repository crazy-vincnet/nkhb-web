## 2024-06-11 - [React Rendering Anti-Pattern in Dynamic Pages]
**Learning:** In page-level wrapper components like `Home.tsx` that use a `.map` loop over layout data to dynamically render heavy child section components, defining event handler functions inline within the loop (e.g. `onOpenModal = () => {...}`) causes massive cascading re-renders. Every time `Home.tsx` renders (even on simple route changes or state updates), it creates new function references for all event handlers. If the heavy child sections (`Background.tsx`, `Composition.tsx`, `Guide.tsx`) are not wrapped in `React.memo()`, they will all unnecessarily re-render because React sees the new prop references as a change.
**Action:** Always extract such event handlers out of the render loop and wrap them with `useCallback`. Furthermore, child section components rendered in mapping loops should be wrapped with `React.memo()` to effectively bail out of rendering if only parent state/props unrelated to them changed.
