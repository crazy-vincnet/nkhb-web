## 2026-05-12 - [Unnecessary Full-Page Re-renders on Modal Toggle]
**Learning:** In `src/public/pages/Home.tsx`, modal state (`isArticleModalOpen`, etc.) is managed at the top level. Opening a modal changes this state, which triggers a re-render of the entire `Home` component and all of its child section components (`Hero`, `Background`, `Composition`, `Schedule`, etc.). This causes unnecessary re-renders of heavy, mostly static components. Passing inline functions to these components prevents React from optimizing them natively.
**Action:** Always wrap event handler props with `useCallback` at the page level and apply `React.memo` to the child section components. This specific combination is required because `React.memo` alone won't work if new function instances are passed as props on every render.
