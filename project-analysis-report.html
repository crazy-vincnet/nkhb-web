NKHB Web 통합 명세서 및 어드민 가이드

시스템 아키텍처 분석부터 운영자를 위한 상세 매뉴얼까지

1. 시스템 아키텍처 분석

기술 스택 & 구조

  • Frontend: React 18 (Multi-entry MPA)
  • Backend: Supabase (PostgreSQL, Auth, Storage)
  • I18n: 동적/정적 데이터 병합 커스텀 시스템
  • Design: TailwindCSS 기반 디자인 토큰 관리

핵심 로직 특이사항

  • 실시간 테마 엔진: CSS 변수를 통한 다이나믹 브랜딩.
  • 이미지 최적화: Canvas API를 통한 클라이언트 사이드 WebP 압축.
  • 보안: 데이터베이스 레벨의 RLS 정책 기반 접근 제어.

2. 어드민 페이지 사용 설명서

운영자가 효율적으로 사이트를 관리하기 위한 핵심 매뉴얼입니다.

📍 대시보드 | ✍️ 소식 관리 | 🎙️ 오디오 관리 | 🌐 시각적 편집 | 📋 편지 관리 | 📈 SEO 설정

✍️ 게시글 및 소식 관리 (Posts)

Tiptap 에디터를 사용하여 공지사항이나 뉴스를 작성합니다. 한국어와 영어를 동시에 입력하여 글로벌 대응이 가능합니다.
1
새 소식 작성: "새 소식 작성" 버튼을 눌러 에디터로 진입합니다.
2
자동 번역 활용: 한국어 내용을 작성한 후 "자동 번역(DeepL)" 버튼을 누르면 영어 필드가 즉시 채워집니다.
3
미디어 삽입: 유튜브 URL이나 이미지를 에디터에 직접 삽입하여 풍부한 콘텐츠를 구성할 수 있습니다.

🌐 시각적 편집기 사용법 (Visual Editor)

홈페이지와 About 페이지의 텍스트, 이미지, 스타일을 실시간으로 확인하며 수정합니다.
  • PROPERTIES 탭: 현재 선택된 요소의 텍스트와 개별 스타일(글자색, 여백 등)을 수정합니다.
  • THEME 탭: 사이트 전체의 강조색(Accent), 기본 테두리 둥글기, 섹션 간격 등을 일괄 제어합니다.
  • 모바일 미리보기: 우측 상단의 스마트폰 아이콘을 눌러 모바일 기기에서의 노출 상태를 즉시 점검합니다.
  • 히스토리: Ctrl+Z (되돌리기)와 Ctrl+Shift+Z (다시실행)를 지원합니다.

🎙️ 오디오 및 방송 관리 (Audio & Schedule)

샘플 듣기 관리

  • 오디오 파일을 업로드하면 WebP/MP3 압축이 자동 적용됩니다.
  • 드래그 앤 드롭으로 사용자에게 보여질 재생 목록 순서를 변경할 수 있습니다.

방송 시간표 업데이트

  • 주파수와 요일별 시간을 테이블 형태로 수정합니다.
  • 수정 즉시 퍼블릭 사이트의 하단(Footer) 및 상세 페이지에 반영됩니다.

📋 희망의 편지 관리 (Letters)

  • 상태 관리: 접수된 편지를 대기, 승인, 보관 상태로 변경하여 관리합니다.
  • 개인정보 보호: 접수된 작성자의 위치와 이메일 정보를 확인하고 필요시 삭제하거나 아카이브할 수 있습니다.

3. 운영 팁 및 주의사항

💡 운영 팁

  • 이미지 업로드: 가급적 Visual Editor나 Audio 관리자에서 제공하는 업로드 도구를 사용하세요. 자동으로 용량이 최적화되어 서버 비용을 절감합니다.
  • 번역 검토: DeepL 자동 번역은 매우 정확하지만, 고유 명사(방송 프로그램 이름 등)는 직접 검토하는 것이 좋습니다.

⚠️ 주의사항

  • 저장 버튼: 시각적 편집기(Visual Editor)는 "저장" 버튼을 누르기 전까지는 DB에 반영되지 않습니다.
  • GrapesJS: 동적 페이지를 만들 때 너무 복잡한 HTML 구조는 로딩 속도에 영향을 줄 수 있습니다.
